# Навык: Анализ данных

## Протокол анализа (выполняй последовательно)

### Шаг 1: Профилирование данных
- Размер датасета (строки × столбцы)
- Типы данных каждого столбца
- Пропуски: количество и процент по каждому столбцу
- Уникальные значения для категориальных переменных

### Шаг 2: Поиск аномалий (ДО любых гипотез)
- Статистические выбросы (IQR или z-score)
- Временные аномалии (резкие скачки/провалы)
- Логические противоречия (дата возврата раньше даты покупки)
- Дубликаты

### Шаг 3: Описательная статистика
- Распределения ключевых числовых переменных
- Группировки по ключевым сегментам

### Шаг 4: Поиск паттернов
- Когортные паттерны (если есть временные данные)
- Сегментные различия
- Неочевидные корреляции между переменными, которые обычно не сравнивают
- Simpson's Paradox — проверь, не скрывает ли агрегация противоположные тренды в подгруппах

### Шаг 5: Executive Summary
- 3 ключевых вывода (от самого важного к менее важному)
- 1 неочевидный инсайт (что нельзя было угадать без данных)
- 1 рекомендация к действию
- 1 вопрос для дальнейшего исследования

## Gotchas
- НЕ начинай с гипотез — сначала посмотри на данные "слепо"
- Проверяй знаменатели: разные знаменатели → разные числа (например, "refund rate" от всех клиентов vs от платящих)
- Маленькие выборки: если в сегменте <30 наблюдений, не делай уверенных выводов
- Сезонность: сравнивай с аналогичным периодом прошлого года, не только с предыдущим месяцем
- Survivorship bias: те, кто остался — не репрезентативны для всех

## Оформление отчёта

Каждый завершённый анализ оформляется в виде самодостаточной папки с отчётом, графиками и скриптом. Структура — рекомендованная, можно адаптировать под свои конвенции.

### Рекомендуемая структура папки

```
{report_dir}/
├── report.md              # Итоговый отчёт (с YAML frontmatter)
├── R1_chart_name.png      # Графики (нумерация R1, R2, ...)
├── R2_chart_name.png
├── analysis.py            # Скрипт анализа и генерации графиков
└── processed.csv          # Обработанные данные (опционально)
```

Пример пути: `analysis/sales/cohort_retention_2026-05-07/`.

### YAML frontmatter в report.md

```yaml
---
date: YYYY-MM-DD
topic: Краткое описание
type: analysis | report | competitive | strategic | note
status: draft | review | final | archived
tags: [tag1, tag2]
related: [path/to/related.md]
executive_summary: |
  2-3 предложения с ключевым выводом
---
```

### Структура отчёта (report.md)

1. **Заголовок** + метаданные (дата, источник данных, период/когорты)
2. **Executive Summary** — самодостаточные 2-3 абзаца с ключевыми находками и цифрами
3. **Нумерованные секции** — каждая: текст → таблица → график → интерпретация
4. **Выводы** — нумерованные, ключевая мысль **жирным**
5. **Методология** — источники, определения метрик, ограничения, ссылка на скрипт

### Графики — ОБЯЗАТЕЛЬНЫ

<important>
Каждый анализ данных ДОЛЖЕН содержать графики в формате PNG.
Для каждой ключевой таблицы в отчёте — создай визуализацию.

Процесс:
1. Напиши Python-скрипт (analysis.py) с pandas + matplotlib/seaborn
2. Скрипт читает исходные данные, строит графики, сохраняет в PNG
3. ВЫПОЛНИ скрипт — графики должны быть реальными файлами
4. В report.md вставь ссылки: `![Описание](R1_chart_name.png)`
</important>

Требования к графикам:
- Формат: PNG, dpi=150
- Нумерация: `R1_`, `R2_`, `R3_`... (соответствует номеру секции)
- Подписи осей и заголовок — на языке отчёта
- `plt.tight_layout()` перед сохранением
- Цветовая палитра: seaborn `muted` или `colorblind`

### Скрипт (analysis.py)
- Комментарий вверху: что делает, входные/выходные данные
- Должен работать автономно: `python analysis.py`
- Зависимости: `pandas`, `matplotlib`, `seaborn`

### Форматирование чисел
- **Деньги**: разделитель тысяч + явная валюта (`47 196 ₽`, `1,250.00 USD`, `980 EUR`)
- **Проценты**: один знак после запятой (`12.7%`)
- **Большие количества**: целые с разделителем тысяч (`30 740` или `30,740`)
